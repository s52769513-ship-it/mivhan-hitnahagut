# אבטחה — תיקון האזהרות של Supabase

התראת ה-Supabase שהופיעה ("שולחן נגיש לציבור" / `rls_disabled_in_public`) פירושה
שכל מי שיש לו את כתובת הפרויקט יכול לקרוא, לערוך ולמחוק את **כל** הנתונים —
כולל נתוני מטופלים. במהלך הבדיקה התגלתה גם בעיה חמורה עוד יותר: **מפתח
`service_role` היה מוטמע בקוד הצד-לקוח** (`index.html`). מפתח זה עוקף לחלוטין
את כל מנגנוני האבטחה, כך שכל מי שפתח את קוד המקור של הדף קיבל שליטה מלאה על
מסד הנתונים.

## מה כבר תוקן בקוד (בוצע ונדחף)

- מפתח ה-`service_role` **הוסר** מ-`index.html`.
- כל הקריאות שהשתמשו בו (ניהול משתמשים, אחסון קבצים, הגדרות) הועברו לעבוד
  עם המפתח הציבורי (`anon`) + אסימון ההתחברות (JWT) של המשתמש המחובר —
  גישה הכפופה למדיניות ה-RLS.

## מה צריך לעשות ידנית (חובה — רק לך יש גישה לזה)

### 1. להפעיל RLS במסד הנתונים
פותחים את Supabase → **SQL Editor** → **New query**, מדביקים את כל התוכן של
הקובץ [`supabase/enable-rls.sql`](supabase/enable-rls.sql) ולוחצים **Run**.
פעולה זו מפעילה RLS על כל הטבלאות, חוסמת גישה אנונימית, ומוסיפה את מדיניות
האחסון הדרושה כדי שהעלאת מסמכים תמשיך לעבוד.

> חשוב: יש להריץ את קובץ ה-SQL כדי שהעלאת/צפייה במסמכים תעבוד לאחר הסרת מפתח
> ה-`service_role`. עד שמריצים אותו — פעולות האחסון עלולות להיכשל.

### 2. להחליף (rotate) את מפתח ה-service_role שנחשף
המפתח הישן דלף (הוא היה בקוד ובהיסטוריית ה-git), ולכן חייבים לבטל אותו:
Supabase → **Project Settings → API → JWT / API keys →** לחצן **Roll**/**Reset**
עבור `service_role` (ובמידת האפשר גם `anon`). זו הדרך היחידה לוודא שהמפתח הישן
כבר לא תקף — שינוי הקוד לבדו אינו מבטל מפתח שכבר פורסם.

## המלצות המשך (לא דחוף, לא חוסם)

- **דלי המסמכים (`patient-files`) מוגדר ציבורי** — כל מי שיש לו קישור ישיר
  יכול לפתוח מסמך מטופל. כדאי להפוך אותו לפרטי ולעבור לקישורים חתומים
  (signed URLs). זו החלפה גדולה יותר כי היא משפיעה על קישורים קיימים.
- **הגבלת גישה לפי הרשאות בצד השרת** — כרגע כל משתמש *מחובר* יכול (טכנית)
  לגשת לכל הרשומות דרך ה-API; הסינון לפי מטופלים/מוסדות מתבצע בצד הלקוח בלבד.
  אפשר להדק זאת בהמשך עם מדיניות RLS מפורטת יותר (לפי `user_patients` /
  `user_institutions`). זהו שיפור, אך המצב הנוכחי כבר עדיף בהרבה על גישה
  ציבורית לחלוטין.
